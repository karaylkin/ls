# Числовой покер (LS)

Многопользовательская онлайн‑игра: угадай число + покерные ставки. Реализация строго по ТЗ
(`TZ_poker_quiz_v2.md`). Ведущего‑человека нет — вопросы, подсказки и правильный ответ выдаёт
сервер из базы `questions.json` (107 вопросов).

## Запуск

```bash
npm install
npm start            # сервер на http://localhost:3000 (отдаёт и WebSocket, и клиент)
```

Переменная окружения `PORT` меняет порт. Откройте `http://localhost:3000` в нескольких вкладках,
чтобы сыграть несколькими игроками.

### Прочие команды

```bash
npm run typecheck    # проверка типов
npm test             # юнит‑тесты побочных банков (все 6 сценариев из 6.4)
node scripts/smoke.mjs   # интеграционный прогон полного раунда двумя ботами
npm run build && npm run serve   # сборка в dist/ и запуск скомпилированного кода
```

## Стек

- **Бэкенд:** Node.js + TypeScript, запуск через `tsx` (без отдельного шага сборки). Real‑time — Socket.IO.
- **Фронтенд:** одностраничное приложение на ванильном JS (без сборки), клиент Socket.IO
  отдаётся сервером с `/socket.io/socket.io.js`. Шрифты — Google Fonts (Bodoni Moda, Cormorant
  Garamond, Jost).
- **Состояние** живёт только в памяти сервера (объект `Room`). БД не требуется.
- **Авторитетный сервер:** клиент шлёт только намерения; сервер валидирует каждое действие и
  никогда не отправляет чужие скрытые ответы до вскрытия (персонализированный `state` каждому).

## Структура

```

src/
  server.ts             express + socket.io, статика, маршрутизация событий
  types.ts              общие типы и дефолтные настройки
  util.ts               коды лобби, id, токены
  game/
    questions.ts        загрузка вопросов, выбор неповторяющегося
    sidepots.ts         алгоритм побочных банков (6.4)
    sidepots.test.ts    тесты на 6 сценариев из ТЗ
    room.ts             конечный автомат раунда + механика ставок + таймеры
    roomManager.ts      реестр комнат, персонализированный broadcast
public/
  index.html / styles.css / app.js   клиент («казино‑нуар»)
  sounds/               сюда кладутся mp3 (опционально)
scripts/smoke.mjs       интеграционный прогон
```

## Деплой

Любой Node‑хостинг (Railway / Render / Fly.io) или VPS за Nginx с TLS. Один процесс отдаёт и
статику, и WebSocket. В проде WebSocket идёт по `wss://` автоматически (клиент использует
относительный origin). Команда запуска на хостинге: `npm start` (или `npm run build && npm run serve`).

## Зафиксированные решения по спорным/краевым случаям

ТЗ в ряде мест оставляет выбор исполнителю — ниже принятые решения.

1. **Никто не вписал ответ** (все авто‑пас по таймеру): раунд считается несостоявшимся, **анте
   возвращается всем**, сразу идёт экран STANDINGS и авто‑старт следующего раунда (раздел 17).
2. **Лимит повышений и олл‑ин** (неоднозначность 3.4): лимит `maxRaisesPerRound` ограничивает
   только добровольные повышения (`raise`). **Олл‑ин всегда легален** в свой ход; если он
   превышает текущую ставку — открывает круг заново (даёт оппонентам право ответить), но **не
   расходует** счётчик повышений. Олл‑ин не больше текущей ставки трактуется как колл.
3. **Закрытие круга** реализовано ровно по 6.3: множество `actedSinceRaise`; любое повышение
   (в т.ч. олл‑ин‑повышение) обнуляет его и кладёт туда только агрессора; круг закрыт, когда все
   дееспособные в множестве и их `roundBet == currentBet`. Если дееспособных ≤ 1 — оставшиеся
   круги проходят без ставок (с короткой паузой для драматургии открытия подсказок/ответа).
4. **Побочные банки** (6.4): слой без претендентов (все его участники спасовали/без ответа)
   **возвращается вкладчикам** этого слоя (невостребованный олл‑ин‑овербет). Неделимый остаток
   при ничьей — игрокам с меньшим номером места.
5. **Вскрытие:** на полном SHOWDOWN раскрываются карты всех не спасовавших участников; карты
   спасовавших остаются скрытыми. При досрочной победе (все спасовали) победитель может нажать
   «Показать», остальные карты скрыты.
6. **Отключения:** в `LOBBY` отключившийся до старта игрок освобождает место. **Во время игры
   место сохраняется** — переподключение по `sessionToken` (хранится в `localStorage`) возвращает
   игрока на то же место с теми же фишками; если он «в ходу», таймер продолжает идти и при
   истечении срабатывает авто‑действие (чек, иначе пас).
7. **Хост:** права ограничены тремя действиями (старт, изменение настроек в лобби, закрытие
   лобби). Раунды сменяются автоматически. При отключении хоста роль переходит первому
   подключённому не выбывшему игроку; при возврате прежнего хоста права не возвращаются.
8. **Анте:** `ante(r) = ante₀ × 2^floor((r−1)/anteDoubleEveryRounds)`, с учётом `anteCap`. При
   дефолтах ряд: 500, 1000, 2000, 4000, … Если фишек меньше анте — игрок вносит остаток и уходит
   в олл‑ин.
9. **Звуки:** активна **заглушка на Web Audio API** (синтез простых тонов на каждое событие из
   раздела 13). Чтобы заменить на реальные сэмплы — положите mp3 с соответствующими именами в
   `public/sounds/`; точка интеграции Howler.js отмечена в `public/app.js` (объект `Sound`).
10. **Стартовый стек / настройки** меняются хостом только в лобби; изменение стартового стека
    сразу применяется к уже сидящим игрокам.
